Этот гайд проходит лабу целиком: вход на платформу, запуск готовой среды GNS3, настройка двух хостов и проверка связности. Делай по шагам, и в конце авто-валидатор покажет зелёный.

Что уже сделано за тебя: роутеры R1 и R2 поднимаются и настраиваются автоматически. На каждом уже есть IP к хосту, IP на линке между роутерами и протокол OSPF. Тебе остаётся настроить два компьютера PC1 и PC3 и убедиться, что пинг между ними проходит.

![Топология OSPF между сайтами](/diagrams/ospf-vlan-topology-frr.svg)

## Часть I. Запуск лабы на платформе

### Шаг 1. Войти на платформу

Открой `http://localhost:3000`. В правом верхнем углу нажми **«Войти»**, на странице входа выбери **«Войти через GitHub»** и авторизуйся. После входа в углу появится твой логин.

### Шаг 2. Открыть лабораторную

В шапке нажми **«Лабы»** и в каталоге выбери **«OSPF между сайтами (FRR Linux routers)»**. Откроется описание с топологией, IP-планом и критериями готовности.

![Страница лабы](/labs/ospf-vlan-lab-frr/screenshots/s02-lab-page.webp)

Изучи IP-план — адреса для PC1 и PC3 понадобятся на шаге 6 и 7.

### Шаг 3. Запустить среду

Нажми **«Запустить»**. Появится диалог подтверждения, нажми **«Запустить»** ещё раз. Платформа за несколько секунд создаст для тебя выделенную среду GNS3.

Появится диалог доступа с логином, паролем и ссылкой на GNS3. **Сохрани пароль** — полностью он показывается только сейчас. Затем нажми **«Перейти к лабе»**.

### Шаг 4. Страница сессии

Ты на странице сессии. Здесь кнопка **«Проверить»** (авто-валидация), доступы к GNS3, список узлов и статус запуска. Дождись, пока все 4 узла будут в статусе запущено.

![Страница сессии](/labs/ospf-vlan-lab-frr/screenshots/s05-session-page.webp)

## Часть II. Настройка хостов в GNS3

### Шаг 5. Открыть GNS3

Нажми **«Открыть GNS3»** на странице сессии. Если GNS3 попросит логин — введи логин и пароль из диалога доступа (шаг 3). Откроется канвас с топологией: два роутера R1 и R2 посередине, хосты PC1 и PC3 по краям.

![Топология в GNS3](/labs/ospf-vlan-lab-frr/screenshots/s06-gns3-canvas.webp)

Роутеры уже запущены и настроены. Тебе нужны только PC1 и PC3.

### Шаг 6. Настроить PC1

Дважды кликни по **PC1** — откроется консоль VPCS. В приглашении `VPCS>` введи команду, которая задаёт IP-адрес, маску и шлюз, и сохрани конфигурацию:

```
ip 192.168.10.10/24 192.168.10.1
save
```

VPCS подтвердит:

```
Checking for duplicate address...
PC1 : 192.168.10.10 255.255.255.0 gateway 192.168.10.1

Saving startup configuration to startup.vpc
.  done
```

Проверь, что адрес встал, командой `show ip`:

```
PC1> show ip

NAME        : PC1
IP/MASK     : 192.168.10.10/24
GATEWAY     : 192.168.10.1
```

### Шаг 7. Настроить PC3

Так же дважды кликни по **PC3**, открой его консоль и задай его адрес из другой подсети, со своим шлюзом:

```
ip 192.168.110.20/24 192.168.110.1
save
```

## Часть III. Проверка

### Шаг 8. Проверить связность пингом

Вернись в консоль **PC1** и пингуй PC3 на другом сайте:

```
PC1> ping 192.168.110.20
```

Если всё настроено верно, придут ответы. `ttl=62` означает, что пакет прошёл через два роутера:

```
84 bytes from 192.168.110.20 icmp_seq=1 ttl=62 time=0.851 ms
84 bytes from 192.168.110.20 icmp_seq=2 ttl=62 time=0.533 ms
84 bytes from 192.168.110.20 icmp_seq=3 ttl=62 time=0.661 ms
84 bytes from 192.168.110.20 icmp_seq=4 ttl=62 time=0.516 ms
84 bytes from 192.168.110.20 icmp_seq=5 ttl=62 time=0.268 ms
```

Если ответов нет, проверь адреса и шлюзы на обоих PC через `show ip` и подожди 10-15 секунд — OSPF мог ещё не сойтись после старта.

### Шаг 9. Авто-валидация

Вернись на страницу сессии платформы и нажми **«Проверить»**. Валидатор пройдёт по трём шагам и покажет зелёный по каждому:

1. **PC настроены** — у PC1 и PC3 корректные IP и шлюзы.
2. **OSPF Full adjacency** — роутеры установили соседство и обменялись маршрутами.
3. **Связность cross-site** — пинг между сайтами проходит.

![Валидация пройдена](/labs/ospf-vlan-lab-frr/screenshots/s07-validation-passed.webp)

Все три зелёные — лаба пройдена.

## Что дальше

- **Посмотреть конфиг роутера.** Открой консоль R1 в GNS3 и выполни `show running-config` в `vtysh` — увидишь, что именно применилось автоматически: IP на интерфейсах и блок `router ospf`.
- **Посмотреть OSPF-соседа.** На R1 выполни `show ip ospf neighbor` — сосед `2.2.2.2` (R2) должен быть в состоянии `Full`.
- **Эксперимент с отказом.** Положи интерфейс к хосту на R2 и снова прогони валидацию — увидишь красный на шаге связности.
- **Завершить сессию.** На странице сессии нажми **«Завершить»**, когда закончишь.
